One-UI Migration
===

## Purpose

Angular 16 to 20 migration with DDD architecture and SignalStore patterns. Follow One-UI migration standards for MXSecurity project.

## Agents

| Agent | Description |
|-------|-------------|
| migration-executor | Execute migrations: analysis, DDD structure, syntax transforms, build verification |
| migration-reviewer | Review migrations: compliance check, completeness review, QA verification |

## Skills

| Skill | Description |
|-------|-------------|
| one-ui-migration | Core migration rules, DDD architecture, Angular 20 patterns |
| migrate-page | Migrate page from local source with DDD architecture |
| migrate-page-gitlab | Migrate page from GitLab source with DDD architecture |
| full-migration-pipeline | Complete end-to-end migration workflow |
| migration-planning | Plan migrations with superpowers integration |
| migration-lint | Compliance checker with auto-fix capabilities |
| migration-review | Compare source vs target completeness |
| form-extraction | Extract and analyze Angular Reactive Forms |
| compare-i18n-keys | Compare translation keys between codebases |
| check-barrel-files | Check for redundant barrel files |
| generate-qa-test-cases | Generate QA test cases from code |
| verify-legacy-with-qa-testcases | Verify legacy against test cases |
| migration-patterns | Query migration patterns and examples |
| icon-replacement | Find replacement icons for migration |
| ui-layout-guide | UI layout optimization guide |
